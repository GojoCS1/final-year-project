# digital Hospital Management System (dHMS)

A web-based, client-server digital health record system designed to transition medical facility workflows from manual, paper-based processes into a secure, centralized digital architecture. This system optimizes patient data entry, scheduling, and administrative reporting for improved healthcare delivery.

---

## 🚀 Key Features
* **Digital Health Records (DHRS):** Secure creation, updates, and maintenance of patient medical histories, diagnoses, and treatments.
* **User Authentication & Security:** Role-based access control ensuring only authorized hospital personnel (Doctors, Nurses, Admins) can view sensitive data.
* **Appointment Scheduling:** Structured queue management to streamline patient visits and reduce clinic wait times.
* **Optimized Database Architecture:** Relational database back-end designed for high-speed query execution and secure data retrieval.
* **Intuitive UI/UX:** Clean, accessible dashboard interface built with graphic design principles to minimize the system learning curve for hospital staff.

---

## 🛠️ Technologies Used
* **Backend Engineering:** php
* **Frontend Interface:** HTML5, CSS3, JavaScript.
* **Database Management:** MySQL / XAMPP Server Architecture
* **Design Tools:** Professional layout design and consulting implemented for optimal UI/UX.

---

## 📐 System Architecture
The system operates on a robust **Client-Server Architecture**:
1.  **Client Tier (Frontend):** Web browser-accessible user interface designed for hospital endpoints.
2.  **Application Tier (Backend):** Built with php to process core logic, manage sessions, and handle secure data routing.
3.  **Data Tier (Database):** A structured MySQL relational database managing entity relations securely.
